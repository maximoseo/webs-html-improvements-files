CRITICAL WORDPRESS REALITY (READ BEFORE YOU GENERATE)

WordPress strips style blocks and script blocks from page/post content in most editors (Classic, Gutenberg, Elementor HTML widget). WordPress wpautop also injects unwanted br and p tags. So:

RULE 1 — Inline everything visual. Every element MUST have its complete styling as inline style attributes. Do not rely on a style block for any visual rendering. You MAY include a single style block as a bonus fallback, but the template MUST look correct without it.

RULE 2 — Inline handlers only. All interactivity (accordion toggles for FAQ and Table of Contents) MUST use inline onclick handlers. No script blocks. Keep onclick expressions on a single line with no line breaks.

RULE 3 — wpautop-safe structure. No blank lines between elements inside containers. Button content on single lines. Wrap standalone links in divs. Minimize whitespace between tags. Include these two defensive rules inside the optional style block: `.sa-article summary br,.sa-article [data-resp="float-group"] br{display:none !important}` and `.sa-article [data-resp="container"] > p:empty{display:none !important}`.

RULE 4 — Zero external dependencies. No Bootstrap, no jQuery, no Font Awesome, no Google Fonts, no CDN links. Use the system font stack only.
