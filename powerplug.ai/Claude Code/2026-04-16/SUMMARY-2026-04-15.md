## Key Design Decisions

| Decision | Rationale |
|---|---|
| Use Hermes as the structural base | Research synthesis showed Hermes had the strongest brand integrity, correct TOC placement, verified case studies, and the cleanest `<details>` pattern. |
| Floating buttons on the right (not left) | User rule 11 is explicit for LTR articles. TimClaw's left-side choice was a WordPress-theme workaround, not a hard requirement. |
| Reject the current Claude Code hero teal | Off-brand. PowerPlug's palette is navy + green. |
| Restrict case studies to Clalit / Rambam / BGU | Research synthesis flagged these as the only publicly-documented PowerPlug customers. Rule 24 forbids fabrication. |
| Keep images to one illustrative figure | Rule 29 forbids broken images. The author logo serves as the one illustrative figure (object-fit:contain) rather than placeholder stock imagery. |
| Single scoped `<style>` block | WordPress themes frequently strip multiple style blocks. Hermes's consolidation survived multiple themes during prior testing. |
| Use `<details>/<summary>` (no JS accordion) | Collapsible behaviour works in every modern browser with no JS, survives WordPress plugin minifiers, and honours rule 6 + rule 20. |
| 2 Tips + 2 How-To + 3 Did-You-Know | Exceeds the rule 13 minimum (3 total) while keeping visual cadence. Each colour class is used more than once so the reader learns the pattern. |
| Validator enforces the same rules the prompt declares | Two-layer safety. If the model misses a rule, the validator surfaces it in the workflow response before the webhook caller sees a successful payload. |
