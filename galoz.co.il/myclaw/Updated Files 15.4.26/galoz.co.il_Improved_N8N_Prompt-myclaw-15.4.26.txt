WordPress is a hostile environment for complex HTML. These rules ensure the
article survives the WordPress editor, Gutenberg block system, and live
rendering with wpautop enabled.

CRITICAL RULES:
  1. MINIFY ALL HTML — output as a single line or very few lines.
     Any line breaks may be interpreted by wpautop as paragraph breaks.
  2. All CSS via inline style attributes — no <style> blocks.
  3. display:flex for all layouts — NEVER display:grid.

  6. No <h1> anywhere — WordPress controls the H1 for post title.

  11. SVG elements must be fully self-contained inline.

  - wpautop converts double line breaks to <p> tags.
  - Single line breaks are converted to <br> tags.
  - MINIFIED HTML prevents wpautop from misinterpreting structure.
  - Inline styles remain intact when wpautop runs — they are on attributes.

JSON-LD SAFETY:
  - Valid JSON syntax — all strings double-quoted.
  - No trailing commas in JSON objects or arrays.
  - Hebrew text in JSON is fine — use direct Hebrew characters.
  - Wrap in <script type="application/ld+json"> ... </script>
