## 4. H1 ENFORCEMENT RULE — CRITICAL

> **RULE**: The final HTML article MUST contain EXACTLY ONE `<h1>` tag.

- The `<h1>` is the article title, placed BEFORE the reading time bar.
- ALL other headings are `<h2>` or `<h3>` — never `<h1>`.
- If the source article contains multiple `<h1>` tags, DEMOTE all but the first to `<h2>`.
- The N8N article-building node must programmatically enforce this:
  ```javascript
  // In HTML generation node — enforce single H1
  let h1Count = 0;
  html = html.replace(/<h1([^>]*)>/gi, (match, attrs) => {
    h1Count++;
    if (h1Count > 1) return `<h2${attrs}>`;  // Demote extra H1s
    return match;
  });
  html = html.replace(/<\/h1>/gi, (match, offset) => {
    // Replace closing tags for demoted H1s
    return match; // simplified — implement with counter tracking
  });
  ```

## 21. ARTICLE BODY WRITING GUIDELINES

### 21.1 Opening Paragraph Rules
- Start with a strong statement about the importance/relevance of the topic
- Mention the target audience (engineers, integrators, plant managers, etc.)
- Do NOT mention dates, years, or time references
- Do NOT use the company name in the very first sentence
- Length: 2-3 sentences

### 21.2 Section Body Writing
- Each H2 section should have 3-4 paragraphs minimum
- Each paragraph: 3-5 sentences, 50-120 words
- Use concrete technical terminology in Hebrew (maintain technical accuracy)
- Include practical examples from real industrial scenarios
- Reference the galoz.co.il product catalog contextually (not as blatant ads)
- Every section should end with a logical transition to the next topic

### 21.3 Technical Content Standards

When writing about industrial automation topics, include:

**For PLC/Controller sections:**
- Mention specific protocols: Modbus TCP, EtherNet/IP, Profinet, CANopen, DeviceNet, HART
- Mention specific standards: IEC 61131-3, IEC 61131-8
- Mention real-world specs: scan time, I/O count, memory capacity, communication ports
- Mention relevant accessories: expansion modules, communication gateways, HMI panels

**For Sensor/Measurement sections:**
- Mention accuracy classes and measurement ranges
- Mention IP protection ratings: IP54, IP65, IP67, IP68, IP69K
- Mention temperature ranges: -40°C to +85°C industrial grade
- Mention signal types: 4-20mA, 0-10V, PT100, PT1000, Thermocouple K/J/T
- Mention relevant standards: ATEX, CE, UL, FM

**For Communication/Networking sections:**
- Mention specific topologies: star, ring, daisy-chain
- Mention cable categories: Cat5e, Cat6, Cat6A, STP, FTP, S/FTP
- Mention specific devices: managed switches, unmanaged switches, media converters, fiber optic
- Mention cybersecurity considerations for OT/IT convergence
- Mention IIoT platforms: MQTT brokers, OPC UA servers, cloud connectivity

### 21.4 Hebrew Writing Quality Rules

- Use professional, clear Hebrew — not overly formal, not colloquial
- Use active voice where possible: "אנחנו ממליצים" not "מומלץ על ידינו"
- Use second person plural for addressing readers: "אתם", "שלכם", "עבורכם"
- Avoid overly long sentences — max 35 words per sentence
- Use bulleted lists for steps, features, and specifications
- Use numbered lists for processes, sequences, and ranked items
- Technical terms: write in English when standard (PLC, HMI, SCADA, IIoT, Modbus)
- Avoid grammatical errors common in AI Hebrew: wrong gender agreements, incorrect verb conjugations

### 21.5 Section Length Guidelines

| Section Type | Min Words | Max Words |
|---|---|---|
| Intro paragraph | 60 | 120 |
| Each H2 body section | 250 | 500 |
| Tip/How-To/DYK block | 50 | 150 |
| FAQ answer | 60 | 200 |
| CTA body text | 20 | 60 |
| Author bio | 60 | 120 |

## 29. WORDPRESS PASTE GUIDE

When the article is complete, the N8N workflow should:

1. **Strip `<!DOCTYPE html>` and `<html>/<head>/<body>` tags** before WordPress publishing
   - WordPress provides these — only the `<article>...</article>` content block is needed
   - EXCEPT: Include the `<link>` for Heebo font inside `<head>` in the full template file
   - For WP REST API publishing, send only the article body content

2. **WordPress REST API Post Creation**:
   ```json
   {
     "title": "[Article Title]",
     "content": "[Article HTML body — article tag content only]",
     "status": "draft",
     "categories": [category_id],
     "featured_media": 0,
     "meta": {}
   }
   ```

3. **Encoding**: The HTML must be valid UTF-8. Hebrew characters must NOT be HTML-entity-encoded in the content body (keep as raw Unicode Hebrew).

4. **Image handling**: All images use external CDN URLs — no upload to WP Media Library required.
