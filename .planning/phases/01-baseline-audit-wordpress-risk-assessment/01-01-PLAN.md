  <action>
    Read the full 307-line HTML template. For EVERY line, check against the WordPress wp_kses_post() survival rules documented in 01-RESEARCH.md.

    Create the audit directory: `.planning/phases/01-baseline-audit-wordpress-risk-assessment/audit/`

    Write `01-html-template-risk-inventory.md` with these sections:

    **1. Executive Summary** -- total counts by severity, overall risk assessment

    **2. CRITICAL Issues** -- table with columns: ID, Line(s), Pattern, What Happens in WP, Fix Strategy
    Must include at minimum (per WP-01, WP-02):
    - C1: `<style>` block (lines 2-14) -- WordPress strips entire block
    - C2: Every `display:grid` occurrence (grep for `display:grid`) -- layout collapses
    - C3: Every `display:flex` occurrence on `<summary>` elements -- accordion layout breaks
    - C4: Every `display:flex` on product card containers -- vertical stacking lost
    - C5: Every `display:flex` on image centering containers -- centering lost
    - C6: Every `display:flex` on author button containers -- horizontal layout lost
    - C7: Every `display:inline-flex` occurrence -- inline layout lost

    For each: list EXACT line numbers found by grep, not estimated. Cross-reference the CSS property survival matrix from research:
    - `display` is NOT on safe_style_css = CRITICAL
    - `flex-direction`, `align-items`, `justify-content`, `gap`, `flex-wrap` ARE on safe_style_css but useless without display:flex

    **3. HIGH Issues** -- table same format
    Must include (per WP-02, WP-06):
    - H1: All 38 `class="..."` attributes -- dead weight without style block
    - H2: All oritmartin.com references (count exact occurrences with grep)
    - H3: All wixstatic.com image URLs (list each URL with line number)
    - H4: All `display:inline-block` occurrences -- also stripped

    **4. MEDIUM Issues** -- table same format
    Must include (per WP-03, WP-07):
    - M1: All `ontoggle` handlers -- require unfiltered_html
    - M2: All `onmouseover`/`onmouseout` handlers -- require unfiltered_html
    - M3: Floating button positioning conflicts (per WP-05)
    - M4: JSON-LD script blocks -- allowed with unfiltered_html but contain wrong site data
    - M5: `linear-gradient` in background -- verify survival

    **5. LOW Issues** (per WP-04, WP-08)
    - L1: `box-sizing` on article wrapper
    - L2: Redundant `dir="rtl"` on inner elements
    - L3: Missing explicit `text-align:right` on text containers (RTL gap)

    **6. CSS Property Audit Summary** -- table of every unique CSS property found in the template, whether it's on safe_style_css, count of occurrences

    **7. Anchor ID Audit** (per WP-04) -- list every `id="..."` attribute, verify each uses simple ASCII lowercase, note any that might conflict with WordPress auto-generated IDs

    **8. Floating Button Audit** (per WP-05) -- list each floating button element with line number, verify `position:fixed`, `z-index`, `bottom`, `left`/`right` are all on safe_style_css. Note potential theme UI conflicts.

    **9. Hidden Text Risk Audit** (per WP-07) -- for each onmouseover/onmouseout handler, document the base inline style state and confirm all text is visible WITHOUT JavaScript execution

    **10. RTL Alignment Audit** (per WP-08) -- verify `dir="rtl"` on article root, check for physical CSS properties that may conflict with RTL (e.g., `margin-left` vs `margin-right`), check explicit `text-align` on text containers

    Use EXACT line numbers from grep output -- do not estimate or guess line numbers.
  </action>
