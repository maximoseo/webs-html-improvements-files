    **1. Before-State Audit Summary (DEL-07)**

    This section fulfills requirement DEL-07. Document the current state of all three deliverable files:

    **1a. HTML Template Current State:**
    - Total lines, overall structure (article wrapper, sections in order)
    - CSS approach: mix of inline styles + `<style>` block + class attributes
    - Layout approach: flex/grid based (will break in WordPress)
    - Interactive elements: details/summary accordions, onmouseover/onmouseout handlers
    - Image sources: 6 wixstatic.com URLs + 1 Supabase URL
    - Site references: oritmartin.com throughout (not hipsterstyle)
    - JSON-LD: 3 schema blocks after author section

    **1b. N8N Prompt Current State:**
    - Total lines, overall structure (sections, instructions, QA checklist)
    - Key contradictions count (from report 02)
    - oritmartin-specific hardcoded data count
    - N8N expression syntax status

    **1c. N8N Workflow Current State:**
    - Total lines, total nodes, workflow flow description
    - Key bugs count (from report 03)
    - Node connection issues
    - Credential references

    **2. WordPress Rendering Risk Review (DEL-08)**

    This section fulfills requirement DEL-08. Comprehensive wp_kses_post() impact analysis:

    **2a. Consolidated Risk Matrix** -- single table pulling ALL issues from reports 01-04:
    | ID | Severity | File(s) | Issue Summary | wp_kses Impact | Fix Phase |

    Sort by: CRITICAL first, then HIGH, MEDIUM, LOW. Within each severity, sort by number of files affected (cross-file issues first).

    Assign each issue a target Fix Phase:
    - Phase 3 (Template Foundation): display property fixes, style block removal, class cleanup, inline CSS architecture
    - Phase 4 (TOC): TOC-specific accordion and anchor fixes
    - Phase 5 (Products): Product card layout fixes, wixstatic URL replacement
    - Phase 6 (FAQ): FAQ accordion fixes
    - Phase 7 (Author/Social/Floating): Author section, social links, floating button fixes, oritmartin brand replacement
    - Phase 8 (SEO/Hover): JSON-LD fixes, hover state audit
    - Phase 9 (N8N): All prompt contradictions, workflow bugs, expression fixes
    - Phase 2 (Discovery): oritmartin->hipsterstyle URL resolution via Firecrawl

    **2b. The display Property Problem** -- dedicated section explaining the single most impactful issue:
    - 54 occurrences of `display:` in the template
    - `display` is NOT on WordPress safe_style_css whitelist
    - All flex/grid child properties ARE allowed but useless without display:flex/grid
    - Fix strategy options: (A) add display to safe_style_css filter on target WP, (B) redesign all layouts without display property
    - Recommendation: Design for option B (no display) as defensive default, test option A on target WP in Phase 10

    **2c. unfiltered_html Dependency Map** -- list every feature that depends on `unfiltered_html` capability:
    | Feature | Handler Type | Count | Graceful Degradation? |
    - onmouseover/onmouseout (hover effects)
    - ontoggle (accordion animation)
    - script blocks (JSON-LD)
    For each, document whether the feature degrades gracefully (still usable without JS) or breaks.

    **3. Prioritized Change List**

    This is the master driving document for all subsequent phases. Ordered by execution dependency:

    **Priority 1 -- Architectural (Phase 3):**
    - Remove `<style>` block entirely
    - Remove all `class="..."` attributes (dead weight)
    - Replace all `display:flex/grid/inline-flex/inline-block` with WordPress-safe alternatives
    - Establish inline-only CSS architecture

    **Priority 2 -- Content Discovery (Phase 2):**
    - Resolve all oritmartin->hipsterstyle URLs via Firecrawl
    - Verify social profile URLs
    - Discover real product page URLs

    **Priority 3 -- Section Rebuilds (Phases 4-7):**
    - For each section (TOC, Products, FAQ, Author, Floating), list the specific issues from the audit that affect that section

    **Priority 4 -- Cross-Cutting (Phase 8):**
    - JSON-LD schema fixes (remove oritmartin data, fix placement relative to author section)
    - Hover state audit (ensure all base states visible without JS)

    **Priority 5 -- N8N Alignment (Phase 9):**
    - Resolve all prompt contradictions
    - Fix all workflow logic bugs
    - Align prompt instructions with new template architecture

    **Priority 6 -- Validation (Phase 10):**
    - Test actual HTML on target WordPress
    - Verify display property behavior on mahsan.websreport.net
    - Responsive QA

    **4. Open Questions for Phase 2+**

    Pull from research open questions and add any new ones discovered during audit:
    - Target WordPress version on mahsan.websreport.net
    - Whether target WP has custom safe_style_css filter adding `display`
    - HipsterStyle contact info (phone, email) for migration map
    - Maximo LLM API credentials for hipsterstyle deployment
    - Each question: what we know, what's unclear, which phase resolves it
