OUTPUT RULES — EVERY RULE IS MANDATORY:

  1. Output RAW HTML ONLY. Nothing before <article and nothing after </article>.
  2. The output MUST start with: <article lang="he" dir="rtl">
  3. The output MUST end with: </article>
  4. ALL CSS must be inline on every element — style="..."
  5. ZERO <style> blocks anywhere in the output — not in <head>, not inline sections.
  6. ZERO CSS class names — do not write class="..." on any element.
  7. No external stylesheets. No CDN links. No Google Fonts <link> tags.
  8. No JavaScript files or external scripts.
     Exception: JSON-LD schema <script type="application/ld+json"> blocks are
     allowed. Back-to-top button onclick inline handler is allowed.
  9. No markdown syntax — no **, no #, no -, no ```.
  10. No code fences (no ``` anywhere in the output).
  11. No HTML comments (no <!-- --> anywhere in the output).
  12. No H1 tags anywhere in the document.
  13. All HTML attributes use DOUBLE QUOTES — never single quotes.
  14. All tags properly opened and closed. No self-closing div or span.
  15. No emojis anywhere in the output.
  16. No fabricated dates or publication timestamps.
  17. No fake urgency phrases ("מהר לפני שייגמר", "הזמן עכשיו" etc.).
  18. No AI-generated clichés ("ללא ספק", "בעולם של היום", "בסופו של דבר" etc.).
  19. No em-dashes (—) in Hebrew text. Use comma or colon instead.
  20. All font references must be system-safe: 'Arimo', Arial, sans-serif
      Do NOT load Arimo from Google Fonts. Use it as a preferred fallback only.

  IMPORTANT: All JSON-LD must be valid JSON. Double-check that all strings are
  properly escaped and all brackets/braces are balanced.

This section defines the ONLY acceptable implementation for <summary> elements
in both the TOC and FAQ <details> blocks.

RULE: Every <summary> must use position:absolute for its + icon.
RULE: The + icon must be rendered inside a circular border using border-radius:50%.
