## Major Improvements Over Codex Version

### 1. Brand Compliance: Author Logo Fixed (Critical)

**Codex defect:** Author logo rendered as a circular avatar (`border-radius:50%`, fixed 88×88px) — physically cutting off the horizontal PowerPlug wordmark.

**Copilot fix:** Logo now displays as `max-width:180px;height:auto;object-fit:contain` — horizontal format, no circular crop, no fixed pixel dimensions. Logo also now links to `https://powerplug.ai/` (Codex had no link on the logo).

### 2. Floating Button Positioning Corrected (Critical)

**Codex defect:** Both floating buttons (Contact Us + Scroll-to-top) were positioned on the same side (`left:24px`), creating visible overlap on mobile and desktop.

**Copilot fix:**
- Floating Contact Us → `bottom:28px;right:24px` (bottom-right corner)
- Floating Scroll-to-top → `bottom:28px;left:24px` (bottom-left corner)
- No overlap at any breakpoint (verified 375px / 640px / 820px)

### 3. Scroll-to-Top: Appears Only After 300px Scroll

**Codex defect:** Scroll-to-top button was always visible, creating visual clutter before the user had scrolled.

**Copilot fix:** Button starts hidden (`opacity:0;pointer-events:none` via style block). Small passive JS scroll listener at bottom of article shows button only after 300px scroll — immediately useful, not always in the way.

### 4. TOC: Unordered List (No Numbers)

**Codex defect:** TOC used `<ol>` ordered list — introducing auto-numbered items that conflict with the heading text and look awkward in a content-heavy article.

**Copilot fix:** TOC uses `<ul>` unordered list with no list-style numbers. Clean, navigational appearance matches premium content standards.

### 5. Social Links Added to Author Section

**Codex defect:** Author section had no social links — missed conversion touchpoint for readers wanting to follow the brand.

**Copilot addition:** Facebook, LinkedIn, and Twitter/X links with official hover colors (#1877F2, #0A66C2, #1DA1F2) via `onmouseover`/`onmouseout`. Contact button also added to author section.

### 6. Font Updated to Montserrat

**Codex defect:** System font stack used — inconsistent with the WordPress theme's loaded Montserrat.

**Copilot fix:** `font-family:'Montserrat',sans-serif` on article wrapper. Theme-loaded, no external import needed.

### 7. Brand Teal Color Updated

**Codex defect:** Used `#0a8c88` as teal — slightly off from the official `#0fb5b0` brand color.

**Copilot fix:** All teal references now use `#0fb5b0` (per brief). Brighter, more vibrant — better contrast against navy.

## Publisher Caveats

1. **Google Sheets IDs**: Replace `YOUR_GOOGLE_SHEET_ID` in the workflow JSON Grab New Cluster and Update Sheet Status nodes with the actual spreadsheet ID before activating.

2. **WordPress credentials**: The "Publish to WordPress" node requires WordPress API credentials configured in N8N credentials manager for the powerplug.ai WordPress installation.

3. **Anthropic API key**: All AI processing nodes use Anthropic Claude — ensure a valid API key is configured in N8N Anthropic credentials.

4. **Scroll-to-top JS**: The single inline `<script>` in the HTML template will be stripped by some WordPress page builders (Elementor, Beaver Builder) if they sanitize `<script>` tags in post content. If this occurs, the scroll-to-top button can be replaced with an always-visible version (remove opacity:0 default + remove the script) without affecting any other functionality.

5. **Montserrat font**: Template assumes the WordPress theme already loads Montserrat. If the theme changes, add `<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@400;600;700;800&display=swap" rel="stylesheet">` to the theme's `<head>` — do NOT add it inside the article HTML.

6. **Image injection**: The 4 `{{ $json.images.section_N.url }}` expressions require a properly configured N8N image generation or fetch node upstream that populates the `images` object. Verify this node exists and returns the expected JSON structure before running the workflow.

7. **LinkedIn URL**: The LinkedIn URL `https://www.linkedin.com/company/powerplug-ltd/` may redirect to `/company/494877/` — both URLs resolve to the correct page. The `/powerplug-ltd/` slug format is preferred for readability.
